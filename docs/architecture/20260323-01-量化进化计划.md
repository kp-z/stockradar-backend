## 概述

StockRadar 当前是一个 A 股实时异动监控 + 条件选股工具，具备数据采集和信号生成能力，但缺少回测验证、风险管理、模拟交易和策略迭代等量化投资核心环节。本计划将其从"智能盯盘工具"进化为"量化筛选 + 回测 + 模拟交易系统"，不含实盘接入。

**核心决策**：
- 回测引擎：自研轻量 + backtrader 双引擎，设置中切换
- 股票池：全 A 股（~5000 只）
- 存储：SQLite（复用现有方案）
- 不含实盘交易接入

## Phase 2: 回测引擎（双引擎可切换）

### 目标
对策略进行历史验证，计算标准量化指标。提供两套引擎，在设置中切换。

### 双引擎架构

```
backtest/
├── engine.py          # 统一入口：BacktestEngine(engine_type='lite'|'backtrader')
├── engine_lite.py     # 自研轻量引擎：复用 check_condition，~300行
├── engine_bt.py       # backtrader 适配：包装为 bt.Strategy + bt.Cerebro
├── strategy.py        # Strategy 协议 + SchemeStrategy 适配器
└── metrics.py         # 绩效指标计算（两套引擎共用）
```

**engine.py 统一入口**：
```python
def run_backtest(db_path, strategy, config, engine_type='lite') -> BacktestResult:
    if engine_type == 'backtrader':
        return _run_bt(db_path, strategy, config)
    return _run_lite(db_path, strategy, config)
```

**自研轻量引擎 (engine_lite.py)**：
- 逐日回放，直接调用 `check_condition()`
- 优点：与现有代码一致，调试方便，无外部依赖
- 适合：验证现有选股方案的历史表现

**backtrader 引擎 (engine_bt.py)**：
- 将 scheme 条件包装为 `bt.Strategy` 子类
- 使用 `bt.Cerebro` 管理回测生命周期
- 优点：支持更复杂的策略（多时间框架、订单类型、Broker 模拟）
- 适合：进阶用户编写复杂策略

### 核心数据结构

```python
@dataclass
class BacktestConfig:
    start_date: str
    end_date: str
    initial_capital: float = 1_000_000
    stamp_tax: float = 0.0005        # 卖出印花税
    commission: float = 0.00025      # 双边佣金
    slippage_pct: float = 0.001      # 滑点
    max_positions: int = 10
    position_pct: float = 0.10       # 单票最大仓位

@dataclass
class BacktestResult:
    trades: list[dict]
    daily_equity: list[dict]         # [{date, equity, cash, holdings_value}]
    metrics: dict                    # 见下方指标列表
```

### 回测流程（逐日回放）
1. 加载日期范围内所有交易日
2. 每个交易日：策略基于 **T-1 及之前数据** 生成信号（防止未来函数）
3. 以 T 日开盘价 + 滑点执行买卖
4. 扣除佣金和印花税
5. 记录每日权益快照

### 绩效指标 — `backtest/metrics.py`

| 指标 | 说明 |
|------|------|
| 年化收益率 | (最终权益/初始资金)^(250/交易日数) - 1 |
| 最大回撤 | 权益曲线峰到谷最大跌幅 |
| 夏普比率 | (年化收益 - 无风险利率) / 年化波动率 |
| 胜率 | 盈利交易数 / 总交易数 |
| 盈亏比 | 平均盈利 / 平均亏损 |
| 平均持仓天数 | 所有交易的平均持有时间 |
| Calmar 比率 | 年化收益 / 最大回撤 |

## Phase 4: 模拟交易 + 风控

### 目标
虚拟账户跟踪持仓，策略信号自动执行，风控规则约束。

### 数据库 Schema

```sql
CREATE TABLE IF NOT EXISTS paper_accounts (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL REFERENCES users(id),
    name            TEXT NOT NULL DEFAULT 'Default',
    initial_capital REAL NOT NULL DEFAULT 1000000,
    cash            REAL NOT NULL DEFAULT 1000000,
    strategy_json   TEXT DEFAULT NULL,
    risk_rules_json TEXT DEFAULT NULL,
    is_active       INTEGER DEFAULT 1,
    created_at      TEXT DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS paper_positions (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    account_id INTEGER NOT NULL REFERENCES paper_accounts(id),
    code       TEXT NOT NULL,
    shares     INTEGER NOT NULL,
    avg_cost   REAL NOT NULL,
    peak_price REAL NOT NULL DEFAULT 0,
    opened_at  TEXT DEFAULT (datetime('now')),
    UNIQUE(account_id, code)
);

CREATE TABLE IF NOT EXISTS paper_trades (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    account_id  INTEGER NOT NULL REFERENCES paper_accounts(id),
    code        TEXT NOT NULL,
    action      TEXT NOT NULL,
    price       REAL NOT NULL,
    shares      INTEGER NOT NULL,
    commission  REAL DEFAULT 0,
    stamp_tax   REAL DEFAULT 0,
    reason      TEXT DEFAULT '',
    executed_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS paper_daily_snapshots (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    account_id     INTEGER NOT NULL REFERENCES paper_accounts(id),
    date           TEXT NOT NULL,
    equity         REAL NOT NULL,
    cash           REAL NOT NULL,
    holdings_value REAL DEFAULT 0,
    daily_pnl      REAL DEFAULT 0,
    UNIQUE(account_id, date)
);
```

### 风控规则 — `paper/risk_manager.py`

| 规则 | 说明 |
|------|------|
| 单票仓位上限 | 不超过总资金的 N% |
| 最大持仓数 | 同时持有不超过 M 只 |
| 固定止损 | 亏损达 X% 自动卖出 |
| 固定止盈 | 盈利达 Y% 自动卖出 |
| 跟踪止损 | 从最高点回落 Z% 卖出 |
| 最大回撤熔断 | 账户回撤超阈值暂停交易 |
| 每日亏损限额 | 当日亏损超限暂停 |

### 与 scan_loop 集成

在 `server.py` 的 `scan_loop()` 中，获取实时行情后增加钩子：

```python
# scan_loop 内，获取 quotes 之后
if active_paper_accounts:
    await asyncio.to_thread(paper_trading_tick, quotes)
```

`paper_trading_tick()` 执行：策略生成信号 → 风控检查 → 执行买卖 → 收盘时记录日快照。

## 实施顺序与验证

### 实施顺序

```
Phase 1 (数据持久化)
    ↓
Phase 2 (回测引擎) ←→ Phase 3 (策略升级)  [可并行]
    ↓
Phase 4 (模拟交易 + 风控)
    ↓
Phase 5 (绩效面板)
```

### 验证方案

| Phase | 验证方式 |
|-------|---------|
| 1 | 迁移后对比 JSON 和 SQLite 数据一致性；启动后 klines_cache 正常加载 |
| 2 | 用已知策略（如"涨停次日买入持有5天"）回测，人工验证部分交易记录 |
| 3 | 对同一股票，bool 版和 scored 版结果一致性验证 |
| 4 | 创建模拟账户 → 手动买卖 → 验证持仓/资金/交易记录正确 |
| 5 | 前端 tab 切换正常；资金曲线与回测结果数据一致 |

### 关键约束
- 现有实时异动功能完全保留，不受影响
- 新逻辑放在独立模块，server.py 最小改动
- SQLite WAL 模式处理并发读写
- 分钟级数据按需拉取，不全量预加载（控制数据量）
- 全A股历史数据首次拉取需分批进行（约 5000 只 × 500 天），显示进度
- 双回测引擎共用 metrics.py 和 strategy.py，通过 engine.py 统一入口切换
- backtrader 为可选依赖，未安装时自动降级为 lite 引擎
